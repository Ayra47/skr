# Contributing

Thank you for your interest in contributing to SKR.

## Development Philosophy

SKR is built around:
- privacy by design
- end-to-end encryption
- minimalism
- maintainable architecture
- browser-native cryptography

Please keep these principles in mind when proposing changes.

## Pull Requests

- Keep pull requests focused and minimal
- Prefer readability over cleverness
- Avoid unnecessary dependencies
- Discuss large architectural changes before implementation

## Code Style

- TypeScript should remain fully typed
- Prefer modular ES modules
- Follow existing Laravel conventions
- Keep cryptographic logic explicit and auditable

## Security

If you discover a security vulnerability, please do not open a public issue.
Instead, contact the maintainer privately.
