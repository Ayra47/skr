# SKR — Анонимный зашифрованный мессенджер

> Приватный мессенджер, построенный на принципе: приватность — это не функция, а фундамент.

Никакого номера телефона. Никакой почты. Никакого настоящего имени. Только логин, пара ключей и переписки, которые можете прочитать только вы и ваш собеседник.

---

## Что такое SKR?

SKR — это приложение для обмена сообщениями в реальном времени, где **каждое сообщение, файл, фотография и геолокация шифруются end-to-end прямо в браузере** ещё до отправки с устройства. Сервер хранит только зашифрованные данные — он не способен прочитать ваши переписки даже теоретически.

Социальный граф тоже приватен: вы можете добавлять только тех людей, которых действительно знаете. Никаких каталогов пользователей, поиска по имени или случайных запросов в друзья. Единственный способ установить контакт — это **одноразовый код добавления**, который вы передаёте самостоятельно.

## Основные возможности

### Криптография
- **ECDH P-256 обмен ключами** — при регистрации каждый пользователь создаёт собственную пару ключей; публичный ключ отправляется на сервер, приватный никогда не покидает браузер
- **AES-256-GCM шифрование** — каждое сообщение, файл, фрагмент файла, фотография и координаты геолокации шифруются ключом, полученным через ECDH + HKDF, перед отправкой
- **Резервное копирование ключа** — приватный ключ можно экспортировать в зашифрованном виде с использованием PIN-кода и seed-фразы из 12 слов для безопасного использования на других устройствах
- Нулевое серверное расшифрование — сервер архитектурно не способен читать содержимое сообщений

### Система друзей
- Никакого поиска пользователей — невозможно найти или случайно обнаружить другого пользователя
- **10-значные одноразовые коды друзей** — создаются по запросу, действуют 5 минут и могут быть использованы только один раз
- Поддержка QR-кодов для передачи кода при личной встрече
- После использования код мгновенно становится недействительным

### Сообщения
- Доставка сообщений в реальном времени через **Laravel Reverb** (WebSockets)
- Статусы доставки и прочтения
- Индикаторы набора текста
- Emoji picker
- История сообщений с lazy loading

### Передача файлов и фотографий
- Файлы и фотографии шифруются по частям перед загрузкой
- Фотографии автоматически сжимаются (Canvas API, максимум 1280px, JPEG 0.82) для экономии трафика с сохранением оригинала для скачивания
- Полноэкранный просмотр фотографий с **масштабированием и перемещением** (масштабирование колесом к курсору, перемещение перетаскиванием, pinch-to-zoom на мобильных устройствах)
- Для файлов отображаются превью в зависимости от типа

### Передача геолокации в реальном времени
- Передача геопозиции на 5 / 15 / 30 / 60 / 180 минут
- GPS-координаты **шифруются на клиенте** перед каждым обновлением позиции — сервер хранит только последнее зашифрованное состояние
- Встроенная карта Leaflet.js внутри сообщения; нажатие открывает полноэкранный режим
- Также доступна одноразовая отправка местоположения
- Передача автоматически завершается по таймеру или вручную отправителем

### Уведомления
- **Web Push уведомления** через VAPID — работают даже при закрытой вкладке
- Встроенные звуковые и toast-уведомления о новых сообщениях

## Технологический стек
| Layer         | Technology
| Backend       | PHP 8.4, Laravel 13 
| WebSockets    | Laravel Reverb 
| Database      | PostgreSQL@17
| Frontend      | TypeScript, Vite, SCSS, Tailwind CSS 4 
| Cryptography  | Web Crypto API (browser-native, zero dependencies) 
| Maps          | Leaflet.js + OpenStreetMap 
| Push          | Web Push (VAPID) via `minishlink/web-push` 
| QR Codes      | `simplesoftwareio/simple-qrcode`

## Особенности архитектуры

Frontend — полностью типизированное TypeScript-приложение, разделённое на независимые модули (`crypto.ts`, `keys.ts`, `location.ts`, `location-map.ts`, `file-upload.ts`, `file-display.ts`, `messages.ts`, `websocket.ts`, ...) и собираемое через Vite. Никаких фреймворков — только чистые и модульные ES-модули с чётким разделением ответственности.

Слой шифрования использует исключительно **Web Crypto API** — встроенную и аудированную браузерную реализацию. Никакие сторонние криптографические библиотеки не используются и не требуют доверия. Генерация ключей построена по цепочке ECDH → HKDF → AES-256-GCM, с новым IV для каждой операции шифрования.

Backend следует стандартным практикам Laravel: модели Eloquent, route-model binding, form request-валидация, broadcast events для realtime-функциональности и синхронная очередь (`QUEUE_CONNECTION=sync`) для сохранения простоты архитектуры без потери возможностей реального времени.

## Почему Laravel?

Я выбрал Laravel потому, что **глубоко знаю этот фреймворк** — не только как им пользоваться, но и как он устроен внутри. Я тестирую собственный код, пишу feature-тесты, рефакторю ради корректности и поддерживаю кодовую базу с тем же уровнем строгости, который применял бы в production-системах.

Laravel 13 даёт проекту именно то, что нужно: надёжный HTTP-слой, полноценную поддержку WebSocket через Reverb, выразительный Eloquent и зрелую экосистему. Я не борюсь с фреймворком — я использую его так, как он был задуман.

## Запуск локально

Требуются PHP 8.3+, Composer и Node.js 20+.

```bash
git clone <repo>
cd skr
composer install
npm install
cp .env.example .env
php artisan key:generate
php artisan migrate
```
